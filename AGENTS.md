# AGENTS.md

## Project Overview

PCG Toolkit for Unity is an **Editor-only** procedural content generation framework inspired by Houdini SOP nodes. It provides a visual node editor (Unity GraphView) and an AI Agent HTTP API for building procedural geometry pipelines that output FBX, Prefab, Material, and Scene assets. Unity 2022.3.16f1, URP, C# 9.0, .NET Framework 4.7.1.

All code is under `Assets/PCGToolkit/`. Nothing compiles into Runtime builds except `PCGGraphData`, `PCGGraphRunner` (MonoBehaviour), and `PCGExposedParam` in `Runtime/`.

## Build & Test

This is a Unity project. There is no CLI build or test runner outside the Unity Editor.

- **Open in Unity**: Open the project folder in Unity 2022.3.16f1. The .csproj/.sln files are auto-generated by Unity/Rider.
- **Run tests**: Unity Editor > Window > General > Test Runner > Edit Mode tab. Tests use NUnit (`com.unity.test-framework@1.1.33`).
- **Test locations**:
  - `Assets/PCGToolkit/Editor/Tests/` — Editor tests (node tests, cache tests, graph execution, agent integration, selection)
  - `Assets/PCGToolkit/Tests/` — Runtime test assembly (ExpressionParserTests)
- **Test base class**: `NodeTestBase` provides `ExecuteNode<T>()`, `AssertGeometry()`, `AssertValidTopology()`, `CreateTestBox()` helpers.

Tests cannot be run from the command line without Unity's test runner (no standalone NUnit runner).

## Architecture

### Three-Layer Design

| Layer | Purpose | Location |
|---|---|---|
| **Bottom** — PCG Node Library | Atomic geometry operations, each as a `PCGNodeBase` subclass | `Editor/Nodes/{Create,Attribute,Geometry,UV,Distribute,Curve,Deform,Topology,Procedural,Output,Utility,Input}/` |
| **Middle** — GraphView Editor + SubGraph | Visual node editor, graph serialization/execution, SubGraph encapsulation | `Editor/Graph/` |
| **Top** — AI Agent Interface | HTTP/WebSocket server, Skill system, JSON Schema export | `Editor/Skill/`, `Editor/Communication/` |

### Core Data Flow

```
PCGGeometry (Points + Primitives + AttributeStore per level)
    → Nodes transform PCGGeometry
    → PCGGeometryToMesh converts to Unity Mesh only at output stage
```

### Key Types

- **`PCGGeometry`** (`Core/PCGGeometry.cs`): Core geometry data with 4-level attribute system (Point/Vertex/Primitive/Detail) and group system. Analogous to Houdini's Geometry.
- **`PCGNodeBase`** (`Core/PCGNodeBase.cs`): Abstract base for all nodes. Implements `IPCGNode`. Provides `GetParamFloat/Int/Bool/String/Vector3/Color/GameObject()` helpers, `ValidateInputs()`, and `SingleOutput()`.
- **`PCGParamSchema`** (`Core/PCGParamSchema.cs`): Defines ports (input/output) with type (`PCGPortType`), direction, constraints, and `Exposed` flag for HDA-style parameter exposure.
- **`PCGContext`** (`Core/PCGContext.cs`): Execution context with node output cache, global variables, error/warning logging, disk cache toggle, and scene references.
- **`PCGNodeRegistry`** (`Core/PCGNodeRegistry.cs`): Auto-scans all `PCGNodeBase` subclasses via reflection, registers them by name. Used by both search menu and Skill system.
- **`PCGGraphData`** (`Runtime/PCGGraphData.cs`): ScriptableObject storing the serialized node graph.
- **`PCGGraphRunner`** (`Runtime/PCGGraphRunner.cs`): MonoBehaviour component for HDA-style scene usage with exposed parameter overrides.

### Graph Execution

- **`PCGGraphExecutor`**: Synchronous DAG execution via topological sort.
- **`PCGAsyncGraphExecutor`**: Frame-distributed execution (one node per frame) with highlight → execute → show result phases. Keeps Editor responsive.
- **`PCGSilentExecutor`**: Headless execution for API/Agent calls without UI side effects.
- Graphs saved as `PCGGraphData` ScriptableObject `.asset` files.

### Caching (Three-Tier)

| Tier | Location | Lifecycle |
|---|---|---|
| L1 Memory | `PCGContext.NodeOutputCache` | Lost on domain reload |
| L2 Disk | `Library/PCGToolkit/Cache/` (.pcgcache) | Cross-session, not in VCS |
| L3 Asset | `Assets/PCGToolkit/.cache/` | Manual cleanup |

Cache keys are content-addressable (SHA256 of node type + params + input hash). `CacheNode` provides manual cache points.

### AI Agent Communication

- **`AgentServer`** (`Communication/AgentServer.cs`): HTTP server on `localhost:8765`. Accepts JSON POST requests.
- **`SkillRegistry`** (`Skill/SkillRegistry.cs`): Wraps `PCGNodeRegistry` for Skill discovery.
- **`PCGNodeSkillAdapter`** (`Skill/PCGNodeSkillAdapter.cs`): Filters Geometry ports, exposes only scalar parameters.
- **`SkillSchemaExporter`**: Exports JSON Schema per category for AI agent consumption.
- **Graph API actions**: `create_graph`, `add_node`, `connect_nodes`, `set_param`, `execute_graph`, `save_graph`, `get_graph_info`, `delete_node`, `disconnect_nodes`, `delete_graph`, `list_graphs`, `list_nodes`, `execute_skill`, `execute_pipeline`.

### SubGraph System

SubGraphs encapsulate node groups as reusable nodes. A SubGraph `.asset` contains `SubGraphInputNode` and `SubGraphOutputNode` defining the external interface. Data flows through `ctx.GlobalVariables["SubGraphInput.{portName}"]` / `ctx.GlobalVariables["SubGraphOutput.{portName}"]`. SubGraphs use independent `PCGContext`.

### Adding a New Node

1. Create a class in the appropriate `Editor/Nodes/{Category}/` folder inheriting `PCGNodeBase`.
2. Override `Name`, `DisplayName`, `Description`, `Category`, `Inputs`, `Outputs`, `Execute()`.
3. `PCGNodeRegistry` auto-discovers it via reflection — no manual registration needed.
4. Use `PCGParamSchema` to define input/output ports with proper `PCGPortType` and constraints.

### Third-Party Libraries (in `Editor/ThirdParty/`)

- **geometry3Sharp**: Core geometry kernel (half-edge mesh, Boolean, Remesh, Reduce, spatial queries). Converted to/from `PCGGeometry` via `GeometryBridge.cs`.
- **LibTessDotNet**: Polygon triangulation.
- **xatlas**: UV unwrapping and packing.
- **MIConvexHull**: Convex hull, Delaunay, Voronoi.
- **Clipper2**: 2D boolean operations.

## AI Agent Principles

AI Agent is a **tool developer**, not an asset producer. Its deliverable is SubGraph tools (encapsulated parameterized node graphs) for human artists to use. AI must not make aesthetic decisions — it exposes control parameters and handles internal fill work. See `Assets/PCGToolkit/AI_AGENT_GUIDE.md` for the full guideline.
